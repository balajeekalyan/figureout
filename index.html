Multi-LLM orchestration
without the bloat

FigureOut is a lightweight, modular orchestrator for building LLM workflows. Clean, predictable, easy to debug. No hidden abstractions.

Read the Docs View on GitHub 
from figureout import FigureOut, RoleDefinition
import asyncio

concierge = FigureOut(
    llm="openai",
    llm_version="gpt-4o",
    roles={
        "product_search": RoleDefinition(
            prompt="You are a product search specialist.",
            schema='{"results": [...], "summary": str}',
            guideline="queries about finding products",
        ),
    },
)

result = asyncio.run(concierge.run("Find a blue running shoe under $100"))

Why FigureOut?

⚡

Lightweight

No framework overhead. Install only the provider SDK you need via optional extras.

🔀

Multi-Role Dispatch

Classify queries and dispatch to one or more roles in parallel. Set max_roles > 1 to fan out.

🛠️

Tool Use (MCP)

Pass a FastMCP server to enable tool calling. Works across all supported providers.

📐

Structured Output

Every role returns a typed JSON dict matching your schema. No string parsing required.

🔁

Resilient by Default

Configurable retries, token limits, and caching keep your app stable under load.

🧠

Hallucination Control

inject_date automatically grounds the LLM with today's date, reducing stale or fabricated temporal reasoning in responses.

💰

LLM Token Savings

Route classification to a cheaper model via lite_llm_version. Use interpret_tool_response=False to skip redundant LLM rounds and return tool output directly.

📖

No Learning Curve

Define roles with plain Python dicts and strings — no new DSL, no config files, no decorators to learn. If you know Python, you already know FigureOut.

🚀

Prototype to Production

The same code that works in a notebook runs in production. Swap providers, tune retries, and scale without rewriting your application logic.
